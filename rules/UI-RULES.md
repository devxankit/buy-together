# 🚫 AI UI GENERATION RULES (STRICT)

These rules must ALWAYS be followed when generating UI components, pages, or layouts.

The goal is to produce **modern, production-ready, non-generic UI** that looks like a real startup product — NOT an AI-generated template.

---

# 🎯 CORE PRINCIPLES

- UI must look like a **real SaaS/startup product (e.g. CRED, Razorpay, Notion, Airbnb)**
- Avoid generic layouts, boring forms, and default styles
- Focus on **clean hierarchy, spacing, and usability**
- Prioritize **UX clarity over visual noise**

---

# 🎨 DESIGN STYLE RULES

## 1. Visual Style
- Use **minimal, modern, clean design**
- Prefer **soft shadows, subtle borders, rounded corners (12px–20px)**
- Avoid harsh colors and gradients unless purposeful
- Use **consistent spacing system (8px grid)**

---

## 2. Typography
- Use clear hierarchy:
  - Heading: bold, large
  - Subheading: medium
  - Body: normal
- Avoid random font sizes
- Maintain consistent line height and spacing

---

## 3. Colors
- Use limited color palette (primary, secondary, neutral)
- Avoid too many colors
- Use color for:
  - CTA
  - Alerts
  - Status only

---

## 4. Components
- Use reusable UI components:
  - Buttons
  - Cards
  - Inputs
  - Modals
- Avoid inline styling repetition
- Maintain consistency across pages

---

# 📱 UX RULES (VERY IMPORTANT)

## 1. No Generic Forms ❌
- Avoid long boring forms
- Use:
  - Step-based forms
  - Section grouping
  - Smart defaults

---

## 2. Clear CTA
- Each screen must have **ONE primary action**
- CTA must stand out clearly

---

## 3. Visual Hierarchy
- Important content first
- Avoid clutter
- Use spacing instead of borders

---

## 4. Empty States
- Always design:
  - Empty state
  - Loading state
  - Error state

---

## 5. Feedback
- Show:
  - Loading indicators
  - Success states
  - Error messages

---

# 🧠 PRODUCT-SPECIFIC RULES (FOR THIS PROJECT)

## 1. Group Buying Context
- Show:
  - Number of users joined
  - Progress (e.g. 7/10 users)
  - Urgency (e.g. “3 spots left”)

---

## 2. Trust Elements
- Always include:
  - Ratings
  - Verified badges
  - Vendor name

---

## 3. Social Feel
- UI should feel **interactive and alive**
- Show:
  - Avatars
  - Activity indicators
  - Participation signals

---

## 4. Avoid Over-Engineering
- Do NOT add unnecessary:
  - Animations
  - Fancy effects
- Keep it clean and purposeful

---

# ⚙️ CODE QUALITY RULES

- Use functional components
- Keep components small and reusable
- No inline styles unless necessary
- Use Tailwind CSS properly (utility-first, not messy)

---

# 🚫 STRICTLY AVOID

- Generic Bootstrap-like layouts
- Centered plain forms with no structure
- Random colors or gradients
- Overcrowded UI
- Default browser styles
- “AI-looking” symmetry and stiffness

---

# ✅ WHAT GOOD OUTPUT LOOKS LIKE

- Looks like a funded startup product
- Clean spacing and alignment
- Clear CTA and flow
- Thoughtful UX decisions
- Consistent design system

---

# 🔥 FINAL INSTRUCTION

If UI looks generic, boring, or template-like → REJECT and redesign.

Always aim for:
👉 Clean  
👉 Modern  
👉 Real product feel  
👉 High usability
